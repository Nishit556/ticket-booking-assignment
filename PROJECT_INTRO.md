# Project Introduction

## Event Ticketing System - Multi-Cloud Microservices Architecture

This project implements a **cloud-native event ticketing system** deployed across **AWS and GCP**. The application enables users to browse events, register accounts, book tickets, and view real-time analytics. The system is built using a **microservices architecture** with 6 distinct services, featuring event-driven communication via Kafka, auto-scaling with Kubernetes HPAs, and comprehensive observability with Prometheus and Grafana.

**Key Features:**
- Multi-cloud deployment (AWS EKS for services, GCP Dataproc for analytics)
- Event-driven architecture with Apache Kafka
- Serverless ticket generation using AWS Lambda
- Auto-scaling microservices with Horizontal Pod Autoscalers
- Infrastructure as Code with Terraform
- GitOps deployment with ArgoCD
